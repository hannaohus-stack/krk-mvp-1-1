## 1. 이 화면의 역할

Creator Step 3. 사용자가 영양성분을 **직접 입력**하거나 **면제 여부를 확인**하는 화면.

- 소규모 제조업자는 식품등의 표시기준 제5조에 의해 영양성분 표시 면제 가능
- 면제 여부를 모달에서 자가진단 → 결과에 따라 "면제 적용" 또는 "직접 입력" 분기
- 입력값은 Step 4 라벨 미리보기 + PDF 산출물에 반영

---

## 2. 3가지 모드

이 화면은 사용자 선택에 따라 3개 모드로 전환된다.

| 모드 | 진입 조건 | 화면 상태 |
|------|----------|----------|
| `choose` | 최초 진입 (데이터 없음) | 초기 선택 배너만 표시 |
| `input` | "직접 입력하기" 클릭 | 영양성분 입력 폼 표시 |
| `exempted` | 면제 모달에서 "면제 적용" 클릭 | 면제 확인 배너만 표시 |

---

## 3. 화면 구조

```
[Creator 공통 StepBar]
[Page Header] ← Section eyebrow + H1 (Creator.tsx에서 주입)

[A. 진입 배너] ← choose 또는 exempted 모드
  ├ choose 모드: 초기 선택 배너 (면제 확인 / 직접 입력 버튼)
  └ exempted 모드: 면제 적용됨 배너 + 보조 링크

[B. 영양성분 직접 입력 폼] ← input 모드만 표시
  ├ 상단 액션 바 (섹션 레이블 + Coming Soon 버튼)
  ├ 1회 제공량 입력
  ├ 영양성분표 (테이블 형식)
  └ 안내 박스 + 면제 재확인 링크

[C. 면제 진단 모달] ← 모달 오버레이
  ├ 질문 목록 (yes/no)
  └ 결과 영역 + 액션 버튼
```

---

## 4. 컴포넌트별 설계 지침

### A-1. 초기 선택 배너 (choose 모드)

목적: 면제 여부를 모르는 사용자에게 선택지 제공.

```
💡 소규모 사업자는 영양성분 표시가 면제될 수 있습니다.
   식품등의 표시기준 제5조 기준 적용

   [면제 대상 확인하기]  [직접 입력하기]
```

- Lightbulb 아이콘 — amber(`#B07A1A`)
- 타이틀: 13px, semibold
- 서브텍스트: 12px, muted
- 버튼 2개:
  - `"면제 대상 확인하기"` — btn-ghost (모달 오픈)
  - `"직접 입력하기"` — btn-primary → input 모드 전환

---

### A-2. 면제 적용 배너 (exempted 모드)

목적: 면제 적용됐음을 명확히 확인, 다시 바꿀 수 있는 탈출구 제공.

```
✓ 소규모 제조업 면제 적용
  식품등의 표시기준 제5조에 따라 영양성분 표시가 면제됩니다.
  최종 판단은 관할 지자체 또는 식약처에 문의하세요.

  [다시 진단하기]  ·  [직접 입력하기]
```

- Breath/Heritage 계열 초록 배경(`#f0fdf4`) + 초록 border-left 4px
- CheckCircle2 아이콘 — 초록
- 타이틀: 14px, semibold, 초록
- 설명: 12px, muted
- 보조 링크 2개: 11px underline, muted → hover ink

---

### B. 영양성분 직접 입력 폼 (input 모드)

#### B-1. 상단 액션 바

```
영양성분 직접 입력                    [🚀 자동 계산 — Coming Soon]
```

- 좌: uppercase 11px, muted
- 우: 작은 버튼, border, muted — 클릭 시 2.5초 토스트 표시 후 사라짐 (기능 미구현 안내)

#### B-2. 1회 제공량 입력

- 레이블: `"1회 제공량"` — uppercase 11px, muted
- 입력 행 (max-width 220px):
  - 숫자 인풋 (flex-1, 우측 정렬)
  - 단위 select `g / mL` (72px)

#### B-3. 영양성분표 테이블

6개 항목: 열량 / 탄수화물 / └ 당류 / 단백질 / 지방 / 나트륨

**헤더 행**:
- 검정(`ink`) 배경, 흰 텍스트
- `"영양성분표"` — 13px, bold, center
- 1회 제공량 입력 시 `"1회 Xg 기준"` 서브텍스트 — 흰색 55% 투명도

**항목 행 (반복)**:
- 짝수/홀수 배경 미세 구분
- **열량 행**: 높이 강조 (`h-10`), bold, 인풋 넓게 (`100px`), `열량` 아래 구분선 굵게
- **당류 행**: 들여쓰기 (`pl-8`), `└` 아이콘, muted 텍스트, 인풋 `88px`
- 일반 행: 인풋 `88px`
- 단위: 우측 고정, 숫자는 tabular-nums

인풋 스타일:
- 기본: thin border
- 포커스: Breath border + 연한 Breath ring

#### B-4. 안내 박스 + 면제 재확인 링크

```
• 숫자만 입력 가능합니다. 수치는 공인시험기관 분석 결과 기준으로 입력하세요.
                                                          [면제 확인하기 →]
```

- 연한 배경 + thin border 박스 (flex-1)
- 우측: 11px underline 링크 → choose 모드로 전환

---

### C. 면제 진단 모달

**배경**: 반투명 어두운 오버레이 + blur

**모달 컨테이너**: `max-w-[480px]`, 흰 배경, thin border

**헤더**:
```
영양성분 표시 면제 자가진단                              [✕]
```
- border-bottom
- 타이틀 15px semibold / 닫기 버튼 muted → ink

**질문 목록** (NUTRITION_EXEMPTION_CRITERIA 기반, 반복):

각 질문마다:
```
질문 텍스트 (13px, 1.6 행간)
관련 법규 (11px, muted)
[예]  [아니오]
```

버튼 스타일:
- 미선택: 흰 배경, 옅은 border, muted
- 선택 "예": Heritage 배경, Heritage border, 흰 텍스트
- 선택 "아니오": `#B30000` 배경, 빨간 border, 흰 텍스트
- min-height: 44px (터치 타겟)

**결과 영역** (모든 질문 답변 완료 후 표시):

면제 가능:
```
✓ 면제 대상일 가능성이 높습니다.
  최종 판단은 관할 지자체 또는 식약처에 확인하세요.
  [면제 적용하고 다음으로]
```
- 연한 초록 배경 + 초록 border

면제 불가:
```
⚠️ 면제 대상이 아닐 수 있습니다.
   영양성분을 직접 입력해주세요.
   [직접 입력하기]
```
- 연한 빨간 배경 + 빨간 border

## 5. 모드 전환 흐름

```
최초 진입
  → choose 모드

choose → [면제 대상 확인하기]
  → 모달 오픈
  → 모달 내 [면제 적용] → exempted 모드
  → 모달 내 [직접 입력] → input 모드
  → 모달 닫기 → choose 유지

choose → [직접 입력하기]
  → input 모드

exempted → [다시 진단하기]
  → choose 모드

exempted → [직접 입력하기]
  → input 모드

input → [면제 확인하기]
  → choose 모드
```

## 6. 모바일 대응

- 모달: `mx-4` 여백, 전체 높이 스크롤 가능
- 영양성분표: 인풋 너비 유지 (스크롤 허용)
- 버튼 2개 (면제/직접 입력): 작은 화면에서 세로 쌓기 검토

## 7. 데이터 연결 포인트

| UI 영역 | CreatorData 필드 |
|---------|-----------------|
| 1회 제공량 | `servingSize`, `servingUnit` |
| 영양성분 수치 | `calories`, `totalCarbs`, `sugar`, `protein`, `totalFat`, `sodium` |
| 면제 여부 | `nutritionExempted` (boolean) |

**canGoNext 조건 (Creator.tsx)**:
- `nutritionExempted === true` → 통과
- 직접 입력 모드: 6개 항목 중 1개 이상 `!= ''` 이고 `!= '0'` → 통과

---

## 8. 주의사항

- Creator 내부 Step 3 — StepBar 유지
- 면제 진단 모달: `position: fixed` — StepBar 위로 올라오지 않게 `z-index` 관리
- 자동 계산(Coming Soon)은 현재 토스트만 표시, 실제 기능 없음 — 버튼은 있되 비활성화 느낌보다 "예고" 느낌으로
- "최종 판단은 관할 지자체 또는 식약처에 확인"은 면제 배너와 모달 결과 양쪽에 모두 표시
- 인풋은 숫자만 허용 (`type="number"`, `e`, `E`, `+`, `-` 키 차단)
