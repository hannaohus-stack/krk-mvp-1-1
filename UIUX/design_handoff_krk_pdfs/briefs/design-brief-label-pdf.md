# Design Brief — 라벨 PDF 레이아웃

**산출물 위치** `/payment/complete` → "라벨 PDF" 다운로드  
**참조 파일** `src/components/LabelTemplate.html`  
**법규 기준** 식품등의 표시기준 (식약처 고시 제2025-27호)  
**브리프 버전** v1.0 (2026-05-22)

---

## 1. 이 파일의 용도

고객이 Creator Step 1~4에서 입력한 정보를 바탕으로 **결제 후 즉시 다운로드** 받는 라벨 출력용 PDF.  
고객은 이 파일을 인쇄소에 전달하거나 직접 출력해 **실제 제품에 부착**한다.

> **법적 책임 전제**: 이 라벨이 법규에 맞지 않으면 고객이 실제 과태료·행정처분을 받을 수 있다.  
> 아래 법규 기준은 선택이 아닌 필수 반영 항목이다.

---

## 2. 출력 규격

| 항목 | 기준 |
|------|------|
| 페이지 | A4 1장 (210 × 297mm) |
| 라벨 배치 | 전면(Front) + 후면(Back) 나란히, 중앙 정렬 |
| 크롭 마크 | 각 라벨 모서리 4곳 표시 (인쇄소 재단용) |
| 여백 | 페이지 상하좌우 최소 15mm |
| 컬러 모드 | CMYK 권장 (PDF 출력 기준) |
| 폰트 | Pretendard (웹폰트 embed or subset) |

라벨 물리 크기는 고객이 Creator에서 선택한 규격에 따라 달라짐.  
기본 기준: **90 × 60mm (전면)**, **90 × 90mm (후면)**

---

## 3. 전면 라벨 (Front Label) 구성

```
┌──────────────────────────────┐  ← 크롭 마크
│  [식품유형]        [내용량 + 열량] │  ← 상단 좌우
│                               │
│  제품명 (가장 큰 활자)          │
│  제품 부제목 / 특징 한 줄       │
│                               │
│  ┌─────────────────────────┐  │
│  │   제품 이미지 영역        │  │
│  └─────────────────────────┘  │
│                               │
│  [배지 행: 무방부제 / 소량생산] │  ← 선택적
└──────────────────────────────┘  ← 크롭 마크
```

### 전면 필수 표시항목 및 규격

| 항목 | 위치 | 글자 크기 | 비고 |
|------|------|-----------|------|
| 제품명 | 상단 좌측 | 최소 24pt | 가장 눈에 띄게 |
| 식품유형 | 제품명 위 | 최소 8pt | 예: `잼류`, `소스류` |
| 내용량 + 열량 | 우측 상단 | **최소 12pt** | `250g (152kcal)` 형태 — 열량 병기 의무 |

> ⚠️ **현재 LabelTemplate.html 미반영**: 내용량(`250g`)만 표시, 열량 없음 → 반드시 추가

## 4. 후면 라벨 (Back Label) 구성

```
┌──────────────────────────────┐  ← 크롭 마크
│  제품명 (후면 헤더)            │
│                               │
│  [원재료명]                    │  ← 섹션 구분
│  원재료 나열 (알레르기 원료 볼드) │
│                               │
│  [알레르기 유발물질 박스]       │  ← 색 대비 필수
│                               │
│  [소비기한]                    │
│  [보관방법]  개봉 전 + 개봉 후  │  ← 둘 다 필수
│                               │
│  [영양성분표]  (면제 시 문구)   │
│                               │
│  [제조업소]                    │
│  업소명 · 소재지 · 신고번호     │
│                               │
│  [포장재질]                    │
│                               │
│  부정불량식품 신고 1399 ──────  │  ← 의무 문구
│  [바코드] ········ [분리배출마크]│
└──────────────────────────────┘  ← 크롭 마크
```

## 5. 후면 항목별 법규 기준

### 5-1. 원재료명

- 함량 내림차순 정렬
- **알레르기 유발 원료는 원재료명 목록 내에서 굵은 글씨(볼드) 표시 의무**
  - 예: `딸기(국산) 60%, 설탕 30%, **대두** 5%, 펙틴 0.8%`
- 복합원재료는 괄호 안에 구성 원재료 함량 순 병기

> ⚠️ **현재 미반영**: 원재료명 내 알레르기 원료 볼드 없음 → 반드시 추가

### 5-2. 알레르기 유발물질 박스

- 원재료명 목록 **바로 아래** 별도 박스
- 박스 배경색과 본문 배경색 **반드시 색 대비** (예: 황토 배경 + 진한 텍스트)
- 표시 형식: `알레르기 유발물질: 대두 함유` 또는 `해당 없음`
- 공유주방 사용 시: `본 제품은 ○○을 사용한 시설과 같은 공간에서 제조됩니다.` 추가

### 5-3. 소비기한

- 2023년부터 유통기한 → **소비기한** 의무 전환
- 표시 형식: `소비기한: YYYY.MM.DD 까지` (도트 인쇄 병행 권장)
- 또는: `제조일로부터 ○○일`

### 5-4. 보관방법

- **개봉 전** 보관조건 + **개봉 후** 보관조건 모두 표시
- 예: `직사광선을 피해 서늘한 곳에 보관 / 개봉 후 냉장 보관, 1개월 이내 섭취`

> ⚠️ **현재 미반영**: 개봉 후만 표시 → 개봉 전 보관조건 추가 필요

### 5-5. 영양성분표

- 의무 대상: 연매출 1억 원 이상 또는 50인 이상 사업장
- **면제 대상 소규모**: `영양성분 표시 면제 대상 제품` 문구 표시
- 표시 시: 1회 제공량 기준, 열량·탄수화물·당류·단백질·지방·포화지방·트랜스지방·콜레스테롤·나트륨 9개 항목
- 영양강조표시(무가당·저칼로리 등) 사용 시 → 면제 불가, 9개 항목 전부 필수

### 5-6. 제조업소

- **3개 항목 모두 필수**: 제조업소명 + 소재지(도로명 주소) + 신고번호
- 예: `KRK Foods · 서울시 성동구 ○○로 00 · 식품제조가공업 신고번호 제2024-서울성동-0000호`

### 5-7. 포장재질

- 용기·뚜껑 각각 재질 표기
- 예: `용기: PET / 뚜껑: PP`

### 5-8. 부정불량식품 신고 문구 ← 현재 완전 누락

```
부정·불량식품 신고는 국번 없이 1399
```

- **식품 등의 표시·광고에 관한 법률 시행규칙 별표 2 의무 항목**
- 후면 하단, 바코드 위 또는 최하단에 반드시 포함
- 글자 크기: 최소 8pt, 색상은 배경 대비

> ❌ **현재 LabelTemplate.html 완전 누락** — 반드시 추가

### 5-9. 분리배출 마크

- 환경부 고시 2024-170호 의무 도안 사용
- 포장재 재질별 공식 마크 이미지 (SVG) 삽입
- 크기: 가로·세로 각 8mm 이상
- 현재 코드의 `♻ PET` 텍스트는 법적 효력 없음 → 실제 도안 교체 필요

## 6. 글자 크기 기준 (인쇄 pt 기준)

| 항목 | 최소 크기 | 비고 |
|------|-----------|------|
| 일반 표시사항 | **10pt** | 공간 부족 시 9.6pt까지 허용 |
| 내용량 + 열량 | **12pt** | 주표시면 의무 |
| 제품명 | **12pt 이상** | 가장 크게 |
| 식품유형 | **8pt** | |
| 정보표시면 전체 | 장평 90% / 자간 -5% | 100㎠ 미만 시 장평 50% 허용 |

> ⚠️ **주의**: HTML/CSS의 px ≠ 인쇄 pt. 10pt = 약 13.3px (96dpi 화면 기준).  
> 인쇄용 PDF 생성 시 반드시 **pt 단위**로 설계할 것.  
> 현재 LabelTemplate.html의 9~10px는 인쇄 기준 약 7~7.5pt → 미달 가능.

## 7. 동적 데이터 연결 (Creator 입력값 → 라벨)

| 라벨 항목 | Creator 입력 필드 | 비고 |
|-----------|-------------------|------|
| 제품명 | `productName` | |
| 식품유형 | `categories[]` → `CATEGORY_OFFICIAL` 매핑 | 식약처 공식 분류명 사용 |
| 내용량 | `totalWeight` + `unit` | |
| 열량 | 영양성분 데이터 또는 추정값 | 면제 시 별도 처리 |
| 원재료명 | `ingredients[]` (함량 내림차순) | 알레르기 원료 볼드 자동 처리 |
| 알레르기 목록 | `detectedAllergens[]` | |
| 소비기한 | `expiryDate` | |
| 보관방법 | `storage` | |
| 영양성분 | `calories`, `totalCarbs` 등 9개 필드 | 면제 시 `nutritionExempted: true` |
| 제조업소 | `manufacturer` | 소재지·신고번호 별도 필드 필요 |
| 포장재질 | `packagingMaterials[]` | 분리배출 마크 자동 매핑 |

## 8. 2가지 영양성분 상태 처리

### 상태 A — 영양성분 입력 있음
표 형태로 9개 항목 표시 (1회 제공량 기준)

### 상태 B — 소규모 면제 (`nutritionExempted: true`)
```
[영양성분 표시 면제 대상]
연 매출액 또는 총 매출액이 1억 원 미만인 업소에서 제조·가공하는 식품으로
영양성분 표시가 면제됩니다.
```
표를 생략하고 면제 문구로 대체.

## 9. 법규 GAP 수정 요약 (현재 → 목표)

| # | 항목 | 현재 상태 | 목표 |
|---|------|-----------|------|
| 1 | 내용량 열량 병기 | `250g`만 표시 | `250g (152kcal)` |
| 2 | 원재료명 알레르기 볼드 | 볼드 없음 | 알레르기 원료 `<strong>` 처리 |
| 3 | 보관방법 | 개봉 후만 | 개봉 전 + 개봉 후 |
| 4 | 부정불량 1399 문구 | **없음** | 후면 하단 의무 추가 |
| 5 | 글자 크기 | px 혼용 | 인쇄 pt 기준 재설계 |
| 6 | 분리배출 마크 | 텍스트(♻) | 공식 SVG 도안 |
