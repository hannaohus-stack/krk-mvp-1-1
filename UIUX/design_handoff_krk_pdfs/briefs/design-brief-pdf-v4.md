# Design Brief — PDF 3종 v4 (긴급 수정 지시)

> **작성일:** 2026-05-19
> **수정 사유:** v3 PDF 출력 결과가 Strategic Brief 락(Lock) 결정사항을 반영하지 않음
> **대상:** `design-brief-pdf-v3.md` 수정 → v4로 전환
> **상태:** 출시 전 반드시 수정 필요 (법적 리스크 + 사용자 오인 가능성)

---

## 🚨 왜 수정이 필요한가

현재 출력된 v3 PDF는 디자인 품질은 우수하지만, **2026-05-18 락(Lock)된 PDF 재설계 결정사항이 반영되지 않았습니다.**

### 발견된 핵심 누락

| PDF | 누락 사항 | 위험도 |
|-----|---------|------|
| PDF ② | "신고 입력 가이드" 명칭 변경 | 🔴 사용자가 "제출용"으로 오해 |
| PDF ② | 정부24/식품안전나라 안내 문구 | 🔴 잘못된 신고 경로 인지 |
| PDF ③ | "krk 라벨 검토 리포트" 명칭 변경 | 🔴 "공식 인증서"로 오인 |
| PDF ③ | "자가 점검 기록" 부제 | 🔴 법적 효력 오해 |
| PDF ③ | 면책 문구 4항목 (공식 인증 아님 명시) | 🔴 단속 시 법적 분쟁 가능 |
| PDF ③ | QR 코드 처리 | 🟡 "진위 확인 가능" 오해 |

### 원인

2026-05-18 PDF 재설계 결정이 Strategic Brief에 락되었으나, v3 디자인 브리프 작성 시점에 Product Vision으로 통합되기 전 정보가 사용됨.

→ **Strategic Brief의 결정사항이 우선합니다.**

---

## ✅ 유지할 것 (변경 없음)

v3 작업에서 잘 된 부분, 그대로 유지:

- 디자인 품질 (레이아웃, 색상, 타이포그래피)
- 서버사이드 생성 방식 (Vercel API Routes + jsPDF)
- 3초 이내 응답 목표
- 5개 카테고리 → 식약처 분류명 매핑 (4번 섹션)
- Tier S/A/B 배지 표시
- Tier B 베타 경고 리본 (잘 만들었음)
- 사업자 유형(식품제조가공업/즉판가공업) 표시
- 12개 법규 3-way 결과 (#006400 / #B8860B / #B30000)
- 카테고리별 분기 처리 (이미 완료)

## 🔴 PDF ② 수정 사항 (`/api/generate-report`)

### 변경 1: 파일명

```diff
- 품목제조보고서.pdf
+ krk_품목제조보고_입력가이드.pdf
```

### 변경 2: 상단 제목 영역

```diff
- 제목: "품목제조보고서"
- 영문 부제: "NOTIFICATION OF FOOD PRODUCT MANUFACTURE"
+ 제목: "품목제조보고 입력 가이드"
+ 부제: "정부24 / 식품안전나라 온라인 신고 참고용"
+ 헤더 우상단 라벨: "PDF·02 입력 가이드" (현재 "PDF·02 품목제조보고서")
```

### 변경 3: 본문 상단에 안내 문구 블록 추가 (필수)

본문 표 위에 다음 안내 블록 추가:

```
┌──────────────────────────────────────────┐
│ 📌 입력 가이드 안내                          │
│                                          │
│ 본 문서는 식품위생법 시행규칙 별지 제43호서식을│
│ 기준으로 krk.team이 입력 편의를 위해          │
│ 자동 작성한 가이드입니다.                     │
│                                          │
│ ⚠️ 실제 품목제조보고는 직접 신고하셔야 합니다:│
│   • 정부24: gov.kr                       │
│     > 식품·식품첨가물의 품목제조보고          │
│   • 식품안전나라: foodsafetykorea.go.kr     │
│   • 오프라인: 관할 시군구청 위생과 방문        │
└──────────────────────────────────────────┘
```

**디자인 스펙:**
- 배경: 옅은 회색 (#F5F5F5) 또는 옅은 노란색 (#FFF8E1)
- 테두리: 1px solid #E0E0E0
- 패딩: 12px
- 폰트: 본문보다 작게 (9pt 권장)
- 위치: 제목/일자 영역 바로 아래, 데이터 표 위

### 변경 4: 푸터 안내문 수정

```diff
- "본 보고서는 krk.team에서 출력된 것이며..."
+ "본 가이드는 krk.team 자동 작성 가이드입니다.
+  실제 신고는 정부24(gov.kr)에서 진행해주세요.
+  본 문서는 식품의약품안전처 공식 문서가 아닙니다."
```

### 변경 5: Tier 배지 텍스트 (선택)

현재 "Tier S/A · 정식 검토" 배지가 있는데, 의미가 "정식 신고 가능"으로 오해될 수 있음. 다음으로 변경 권장:

```diff
- "Tier S/A · 정식 검토"
+ "Tier S/A · 표시기준 검토 완료"

- "Tier B · 베타 검토"
+ "Tier B · 베타 검토 (참고용)"
```

### 유지할 것 (PDF ②)

- 데이터 표 (품명, 식약처 분류명, 원재료 등 모든 필드)
- 식약처 분류명 자동 매핑 로직 (잘 작동함)
- 보고서 번호 (KRK-MFR-...)
- 작성일 표시
- 좌우 2단 레이아웃

## 🔴 PDF ③ 수정 사항 (`/api/generate-cert`)

### 변경 1: 파일명

```diff
- 검토완료인증서.pdf
+ krk_라벨검토리포트.pdf
```

### 변경 2: 상단 제목 영역 (가장 중요)

```diff
- 한글 제목: "KRK 식품 라벨 검토 완료 인증서"
- 영문 부제: "CERTIFICATE OF LABEL COMPLIANCE REVIEW"
- 헤더 라벨: "PDF·03 COMPLIANCE"

+ 한글 제목: "krk 라벨 검토 리포트"
+ 한글 부제: "사업자 자율 점검 기록"
+ 영문 부제: "LABEL REVIEW REPORT — SELF-AUDIT RECORD"
+ 헤더 라벨: "PDF·03 자율 점검 기록"
```

⚠️ **중요:** "인증서(Certificate)" 단어를 모든 위치에서 제거. "리포트(Report)" 또는 "기록(Record)"으로 통일.

### 변경 3: 면책 문구 — 푸터 작은 글씨에서 → 본문 상단 명확한 블록으로

**현재 문제:** 면책 문구가 8pt 작은 글씨로 푸터에 1줄만 표시됨. 사용자가 보지 못함.

**수정안:** 본문 상단(검토 결과 표 위)에 명확한 블록으로 표시.

```
┌──────────────────────────────────────────┐
│ ⚠️ 중요 안내 — 반드시 읽어주세요              │
│                                          │
│ 본 리포트는 krk.team 시스템이 입력된 정보를   │
│ 기준으로 자동 검토한 결과이며, 아래 사항을    │
│ 명확히 안내합니다:                          │
│                                          │
│ 1. 본 리포트는 식품의약품안전처 또는 관할     │
│    시군구청이 발급한 공식 인증서가 아닙니다.   │
│ 2. 본 리포트는 법적 효력이 없습니다.         │
│ 3. 식품 표시기준 준수의 최종 책임은           │
│    사업자에게 있습니다.                      │
│ 4. 법규 개정 시 재검토가 필요합니다.          │
│                                          │
│ 본 리포트는 사업자의 자율 점검 노력 기록       │
│ 용도로만 활용하시기 바랍니다.                 │
└──────────────────────────────────────────┘
```

**디자인 스펙:**
- 배경: 옅은 빨간색 (#FFF5F5) 또는 옅은 노란색 (#FFF8E1)
- 테두리: 2px solid #B30000 (또는 #B8860B)
- 패딩: 16px
- 폰트: 본문 크기 (10pt 이상)
- 위치: 제목/일자 영역 바로 아래, 12개 법규 검토 표 위
- 이모지 ⚠️는 옵션

### 변경 4: QR 코드 처리

**현재:** 우하단에 QR 코드 표시, 의미 불명

**수정안 (2가지 중 선택):**

#### 옵션 A: QR 코드 완전 제거 (권장)
- 우하단 QR 영역 삭제
- 대신 일련번호 표시: `자율 점검 기록 번호: KRK-20260519-3148`

#### 옵션 B: QR 코드 유지, 설명 변경
- QR 코드 아래 텍스트: ~~"진위 확인"~~ → "자율 점검 기록 일련번호"
- QR 링크는 외부 진위 확인 페이지가 아닌, 단순 기록 조회 페이지로 연결 (또는 작동 안 함 명시)

### 변경 5: Tier 배지 텍스트

```diff
- "Tier S/A · 정식 검토"
+ "Tier S/A · 자율 점검 완료"

- "Tier B · 베타 검토"
+ "Tier B · 자율 점검 (베타, 참고용)"
```

### 변경 6: 푸터 면책

```diff
- "본 인증서는 krk 자동 검토 시스템의 결과이며 법적 효력을 보장하지 않습니다." (8pt)
+ "krk 라벨 검토 리포트 — 사업자 자율 점검 기록 (법적 효력 없음)" (8pt)
+ 푸터 URL: krk.team/verify/{id} 또는 제거
```

### 유지할 것 (PDF ③)

- 12개 법규 3-way 결과 표 (R01~R12)
- 결과 컬러 (#006400 / #B8860B / #B30000)
- SUMMARY 통과/주의/위반 카운트
- Tier B 베타 경고 리본 (이미 잘 만들었음)
- 검토 일시 표시
- 보고서 번호
